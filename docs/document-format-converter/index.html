Document Format Converter

This page provides information on Document Format Converter from XML <-> JSON/JSON-LD converter.

Introduction:

The previous version of EPCIS standard i.e until EPCIS 1.2 events were generated only in the XML format. However, from EPCIS 2.0 events can be generated and transmitted in either XML or JSON/JSON-LD format. As EPCIS standard has been adopted by various industry sectors such as automotive, food, healthcare, retail, and many more, there is a need to convert the large EPCIS event document from either XML -> JSON/JSON-LD or JSON/JSON-LD -> XML format. This conversion can be useful for organizations to exchange data easily, maintain a single standard format throughout the application, and for various other convenience purposes. Although there are many open-source ready-to-use tools available online and offline that offer these format conversions, they may not be able to convert the EPCIS events accurately. During the conversion, some EPCIS attributes such as User Extensions, Sensor Elements, etc need to be handled carefully and according to standard. Therefore, until now, there is no converter that effectively and efficiently converts the large EPCIS event document from XML <-> JSON/JSON-LD format.

OpenEPCIS wants to support the EPCIS community by providing means to convert the EPCIS events from one format to another with ease. The developed utility aimed to convert the bulk EPCIS events effectively and efficiently. The application provides a single solution for converting EPCIS events irrespective of the industry, use case, or supply chain system. It reduces the time and effort for the various organizations as each of them does not have to develop the conversion tool. Also, maintain and incorporate modifications according to the different release versions of EPCIS. The converted EPCIS events conform to the current version of the EPCIS, i.e. EPCIS 2.0.

Usage:

Depending on their expertise and comfort, users normally utilize the application in various ways. Therefore, it is essential to provide applications in different forms so that users may pick the best option. Hence, OpenEPCIS document format converter application has been provided in following forms:

  • Web application: The tool is operational and accessible as a web application where based on the requirement either XML or JSON/JSON-LD EPCIS document can be directly converted into other format. The application is available at: https://tools.openepcis.io/openepcis-ui/Documentconverter.

  • API endpoint: The application has been designed such that users may submit requests using their documents as arguments to the services API endpoint and receive back the converted format in response. The Swagger-UI is an organized or standardized approach for leveraging the API that may be used to access the application. The Swagger-UI is available at: https://tools.openepcis.io/q/swagger-ui/#/Document%20Converter%20Resource

  • Application code: The Java programming language is primarily used to develop the application. The code is available from the OpenEPCIS GitHub account and can be used directly by users or as dependencies for other projects. The full application code may be found at: https://github.com/openepcis
