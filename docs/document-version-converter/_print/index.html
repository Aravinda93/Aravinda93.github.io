Document Version Converter

This page provides information on Document Version Converter from 1.2 XML <-> 2.0 XML.

Introduction:

In comparison to prior versions, the newest EPCIS version, EPCIS 2.0, offers a number of enhancements, such as the addition of sensor data (How dimension), a new event type (AssociationEvent), a new attribute (PersistentDisposition), etc. There is a need for a mechanism to handle backward and forward compatibility when utilizing the EPCIS standard because older versions of the XML did not support this new information OpenEPCIS has developed a tool that converts back and forth between XML versions 1.2 and 2.0 to address this issue. This makes it easy for any business to use the tool to convert 1.2v XML to 2.0v XML.

Usage:

  • Web application: The tool is operational and accessible as a web application where based on the requirement either 1.2v XML or 2.0v XML EPCIS document can be directly converted into respective format. The application is available at: https://tools.openepcis.io/openepcis-ui/DocumentVersionconverter.

  • API endpoint: The application has been designed such that users may submit requests using their documents as arguments to the services API endpoint and receive back the converted format in response. The Swagger-UI is an organized or standardized approach for leveraging the API that may be used to access the application. The Swagger-UI is available at: https://tools.openepcis.io/q/swagger-ui/#/Document%20Version%20Converter%20Resource

  • Application code: The utility has been developed using Java with Apache Xalan. The code is available from the OpenEPCIS GitHub account and can be used directly by users or as dependencies for other projects. The full application code may be found at: https://github.com/openepcis
