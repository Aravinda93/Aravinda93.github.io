Test Data Generator

OpenEPCIS solution for generating EPCIS test data events for all user cases/supply chain.

Introduction

The problem for various solution providers and user companies adopting EPCIS for their projects is to generate test data for testing purposes (load tests, populating test databases, etc.). OpenEPCIS Test Data Generator was developed to assist the EPCIS community by making it simple to generate test data (e.g. hundreds or thousands of events). Regardless of the industry, use case, or supply chain system, the developed application provides a single solution for creating test events. It saves time and effort for the various organizations because they don’t have to build, maintain, or incorporate new capabilities of EPCIS into their test data production technologies. Based on the information provided by users and their organizational needs, the application can create bulk EPCIS events. These EPCIS events are compliant with EPCIS 2.0, the most recent version of the EPCIS, and events are generated in JSON/JSON-LD format. It also includes the option to generate events and identifiers in both URN and WebURI format.

Usage

  • Web application: The web application is beneficial to users who are new to the EPCIS world as it visually provides all the components of the EPCIS which are segregated based on different dimensions. It is very easy to provide values for necessary fields based on the requirement or supply chain and generate the EPCIS events accordingly. The web application can be fround from here.

  • API endpoint: Users can make use of API endpoint to generate test data events. The required template with all information can be sent to Restful API end point which will be analysed and events are generated. Finally, these events are sent as response to user. Users can access the REST endpoint using Swagger-UI from here.

  • Application code: The utility’s back-end has been developed using Java and web application using Nuxtjs. The code is available at the OpenEPCIS GitHub account and can be accessed directly or as dependencies for other projects. The complete code can be from here.
